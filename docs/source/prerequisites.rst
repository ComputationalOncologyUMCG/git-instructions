Prerequisites
==========================================================================

In our current setup, code is versioned via GitHub. To work with projects within our github environment, you need access to the Computational Oncology UMCG group.

.. _github_account:
Github account
------------------

To do anything, you need a github account. If you do not have one yet, please create one at `github.com <https://github.com/>`_.
It is recommended that you use an email address that you are going to keep (not necessarily UMCG), so that you can still access and showcase your code even after your contract at the UMCG may expire


.. _github_group:
Requesting group access
-----------------------

After creating the GitHub account, you'll have your own personal repository. To work with the Computational Oncology group, you need access to the group, and to a repository to work in.
Ask one of the admins to add you to the group, and what project you will work on. If the project already exists, you will be added to the team for that project, which in turn will also give access to the repository.
(If you are an admin, follow the instructions here :doc:`teams_and_repos_for_admins`)


.. _gitlab_account:
GitLab account (optional)
-----------------------

If you wish to use GitLab for planning, you will need a GitLab account. If you do not have one yet, please create one at `gitlab.com <https://gitlab.com/>`_. It is recommended you use the same email and account name as the GitHub account, so that you can easily link the two accounts. 
Let the admins know that you want to use Gitlab, and for which project. They will then link the Github repo to Gitlab.
(If you are an admin, follow the instructions here :doc:`gitlab_linking_for_admins`)


.. _zenodo_account:
Zenodo account (optional)
-------------------------

If you want to make data for your project publicly available, you can use Zenodo. If you do not have one yet, please create one at `zenodo.org <https://zenodo.org/>`_. It is recommended you use the same email and account name as the GitHub/GitLab account, so that you can easily link the two accounts.
In the case that the data was generated in the group, it might be preferable that the group makes the data public, rather than the individual. In that case, please contact the admins. 


.. _dockerhub_account:
DockerHub account (optional)
-----------------------------

If you want to use DockerHub for automated builds, you will need a DockerHub account. If you do not have one yet, please create one at `hub.docker.com <https://hub.docker.com/>`_. It is recommended you use the same email and account name as the GitHub/GitLab account, so that you can easily link the two accounts.
When the Docker images are part of a continuous integration pipeline, it might be preferable that the group makes the images public, rather than the individual. In that case, please contact the admins.


.. _next_step:

After you have created the required accounts, you can move on to the :doc:`overview` section, which will guide you through the basic usage of git in our group.
